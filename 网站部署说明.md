# 网站部署说明

> 本文用大白话解释「把这个仓库变成一个网站」所做的全部操作，方便日后维护和修改。

---

## 一、整体思路

仓库里的评测都是 **Markdown 文件**（`.md`），本身就是纯文本，人类可读，但浏览器不能直接把它们渲染成好看的网页。

解决方案：使用 **[Docsify](https://docsify.js.org)**。

Docsify 是一个纯前端的工具：它只是一个 `index.html` 页面，当浏览器打开这个页面之后，页面会自动拉取仓库里的 `.md` 文件并实时渲染成 HTML 展示出来。**不需要任何编译或构建步骤**，仓库里的 Markdown 文件完全不用改动。

整个部署流程是这样的：

```
你在本地写好评测 → git push 到 main 分支
       ↓
GitHub Actions 自动触发
       ↓
  生成导航侧边栏（_sidebar.md）
       ↓
  把仓库整体上传到 GitHub Pages
       ↓
网站自动更新，访问者看到最新内容
```

## 二、新增的文件逐一解释

### 1. `index.html` — 网站的"门"

```
仓库根目录/index.html
```

这是整个网站唯一的 HTML 文件，是浏览器访问网站时加载的第一个文件。

它做了三件事：

1. **引入 Docsify 的样式表**（`vue.css`）——让页面好看。
2. **配置 Docsify**——告诉它：
   - 网站叫 "Oneday"
   - 首页是 `Readme.md`
   - 侧边栏从 `_sidebar.md` 读取
   - 搜索框提示文字是中文
   - 翻页按钮显示"上一篇 / 下一篇"
3. **加载 Docsify 引擎**（`docsify.min.js`）以及两个插件：
   - **search**：全文搜索，可以直接搜游戏名
   - **pagination**：文章底部的翻页按钮

---

### 2. `.nojekyll` — 告诉 GitHub"不要用 Jekyll"

```
仓库根目录/.nojekyll（空文件）
```

GitHub Pages 默认会用自己内置的 Jekyll 静态网站生成器处理仓库内容。但我们已经用 Docsify 了，不需要 Jekyll，而且 Jekyll 会忽略以 `_` 开头的文件（比如 `_sidebar.md`），导致网站出错。

放一个空的 `.nojekyll` 文件，GitHub Pages 就会跳过 Jekyll，原样把文件发布出去。

---

### 3. `scripts/gen_sidebar.py` — 自动生成导航栏

```
仓库根目录/scripts/gen_sidebar.py
```

Docsify 的左侧导航栏（侧边栏）需要一个叫 `_sidebar.md` 的文件，里面是导航链接的列表。如果手动维护这个文件，每次新增一篇评测都要手动加一行——很麻烦，而且容易漏。

这个 Python 脚本的作用就是**自动扫描仓库里所有的 `.md` 文件，生成 `_sidebar.md`**。

脚本的主要逻辑：

- 遍历 `Aspark/小众变态测评/` 下的「完成评测」「二游与竞技」「未完成」三个子目录
- 遍历 `Blind-Guess-Senior/Game/`、`Anime/`、`Book/` 下的所有文件
- 对每个文件提取一个干净的**显示标题**：
  - 在 `by-name/`、`by-year/` 等「按名称整理」的目录里，文件名本身就是游戏/动漫名，直接用文件名（去掉 `★★★` 这样的评级后缀）
  - 其他文件则读取文件内容，取第一行非评分的纯文本作为标题

脚本生成的 `_sidebar.md` 只在部署时由 GitHub Actions 生成，**不提交到仓库**（已加入 `.gitignore`）。

---

### 4. `.github/workflows/pages.yml` — 自动部署工作流

```
仓库根目录/.github/workflows/pages.yml
```

这是 **GitHub Actions** 的配置文件，定义了一个「工作流」（Workflow）。可以把它理解成一套自动化脚本，每次满足触发条件时就会在 GitHub 的服务器上自动运行。

**触发条件：**
- 有人向 `main` 分支推送代码（`push: branches: [main]`）
- 或者在 GitHub 网页上手动点击「Run workflow」按钮（`workflow_dispatch`）

**工作流步骤：**

| 步骤 | 作用 |
|------|------|
| `Checkout` | 把仓库代码拉到 GitHub 服务器上 |
| `Generate sidebar` | 运行 `gen_sidebar.py`，生成 `_sidebar.md` |
| `Setup Pages` | 让 GitHub Actions 知道要部署到 GitHub Pages |
| `Upload artifact` | 把整个仓库（含刚生成的 `_sidebar.md`）打包上传 |
| `Deploy to GitHub Pages` | 把打包内容发布到网站，网站随即更新 |

**权限配置：**

```yaml
permissions:
  contents: read   # 允许读取仓库内容
  pages: write     # 允许写入 GitHub Pages
  id-token: write  # 允许使用 OIDC 令牌做身份验证（GitHub Pages 部署需要）
```

---

### 5. `Readme.md` — 网站首页

原来的 `Readme.md` 内容很简单，只有几行英文说明。

现在它被改写成了网站的首页，包含两位作者评测标准的简介，以及说明如何使用导航栏查找评测。访问网站时看到的第一个页面就是它。

---

### 6. `.gitignore` 新增一行

```
# Auto-generated by GitHub Actions – do not commit
_sidebar.md
```

`_sidebar.md` 是每次部署时临时生成的文件，内容随仓库里的 `.md` 文件自动变化。不需要（也不应该）手动维护或提交它，所以加入忽略列表。

## 三、如何启用 GitHub Pages（一次性操作）

> **只需要做一次，之后每次 push 到 main 就会自动更新。**

1. 打开仓库页面，点击顶部的 **Settings**（设置）选项卡
2. 在左侧菜单里找到 **Pages**
3. 在 **Build and deployment** 下面，把 **Source** 改成 **GitHub Actions**
4. 保存

之后把这个 PR 合并到 `main` 分支，工作流就会自动运行，网站地址是：

```
https://blind-guess-senior.github.io/Oneday/
```

## 四、日常使用

### 新增一篇评测后，网站怎么更新？

直接 `git push` 到 `main` 就好了，GitHub Actions 会自动检测到变化并重新部署。网站一般在几分钟内更新。

### 我想修改侧边栏的分组或顺序怎么办？

编辑 `scripts/gen_sidebar.py` 中的 `main()` 函数即可。脚本逻辑清晰，每个分组（Aspark 的完成评测、进行中评测、Blind-Guess-Senior 的游戏/动漫/书籍）都有对应的代码块。

### 我想换网站主题怎么办？

修改 `index.html` 第 7 行的 CSS 链接：

```html
<!-- 当前主题：vue（绿色） -->
<link rel="stylesheet" href="//cdn.jsdelivr.net/npm/docsify@4/lib/themes/vue.css">

<!-- 可替换为其他主题，例如 dark（深色）： -->
<link rel="stylesheet" href="//cdn.jsdelivr.net/npm/docsify@4/lib/themes/dark.css">
```

Docsify 内置主题有：`vue`、`buble`、`dark`、`pure`。
